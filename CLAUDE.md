# CLAUDE.md — repo map

DevDigest: local-first AI pull-request review. This file is the **territory map**
loaded into every session. Per-package details live in `<pkg>/CLAUDE.md` and load
automatically when you touch that package. Procedures live in skills, not here.

## Packages

Not a workspace monorepo: each package has its own `package.json` and lockfile.
Cross-package code is shared via tsconfig path aliases.

| Folder            | Package                    | Stack                                              | Port |
|-------------------|----------------------------|----------------------------------------------------|------|
| `server/`         | `@devdigest/api`           | Fastify 5 + Drizzle/Postgres (pgvector)           | 3001 |
| `client/`         | `@devdigest/web`           | Next.js 15 (App Router) + React 19                | 3000 |
| `reviewer-core/`  | `@devdigest/reviewer-core` | Pure review engine: diff → prompt → LLM → findings | —    |
| `e2e/`            | `@devdigest/e2e`           | Deterministic browser e2e via agent-browser (no LLM) | —    |

`@devdigest/shared` is **not** a separate package — it lives at
`server/src/vendor/shared` (Zod contracts, source of truth) and is vendored into
`client/src/vendor/shared` (read-only copy).

`reviewer-core` is pure (no DB / GitHub / fs); its only side effect is an
injected `LLMProvider`. Consumed as **source** via path alias by both `server`
and the CI runner.

## Runtime

- Node ≥ 22, pnpm ≥ 10 (server and client). `e2e/` and `reviewer-core/` use npm.
- Only Postgres runs in Docker; API and web run on the host.

## Bootstrap

```sh
./scripts/dev.sh        # zero → Postgres + .env + deps + migrate + seed + API + web
```

Flags: `--no-seed`, `--no-client`, `--db-only`, `--help`. Open http://localhost:3000.

DB migrations are **not** applied on boot. Run `pnpm db:migrate` in `server/`.

## Global do-not-touch

- `server/src/db/migrations/**` — generated by Drizzle Kit. Edit `db/schema/`,
  then `pnpm db:generate`. The pgvector extension is enabled by migration `0000`.
- Lockfiles — `pnpm-lock.yaml` (`server/`, `client/`) and `package-lock.json`
  (`e2e/`, `reviewer-core/`). Don't mix pnpm and npm per package — the manager
  split is intentional.
- `client/src/vendor/shared/` — vendored copy of contracts. Edit the source of
  truth in `server/src/vendor/shared/`, not the copy.

## Session capture (engineering-insights skill)

- Before touching code in a package, read its `INSIGHTS.md` and any reference
  the user attached for the task; confirm in one line.
- Before adding a new entry, re-read the target section to avoid duplicates.
- At session end, write an entry only if something non-obvious surfaced that
  isn't already captured. Trivial work → no entry.
