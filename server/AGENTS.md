# server/ — `@devdigest/api` (Fastify 5 + Drizzle/Postgres)

Backend engine. Imports repos/PRs, runs the reviewer pipeline, persists structured
findings. Adapters (LLM, GitHub, git, ast-grep, …) sit behind a DI container.

## Run / verify

| What        | Command                                               |
|-------------|-------------------------------------------------------|
| dev         | `pnpm dev` (tsx watch, `:3001`)                       |
| typecheck   | `pnpm typecheck`                                      |
| unit only   | `pnpm exec vitest run --exclude '**/*.it.test.ts'`    |
| integration | `pnpm exec vitest run .it.test` (needs Docker)        |
| migrate     | `pnpm db:migrate` — NOT auto on boot                  |
| seed        | `pnpm db:seed` — idempotent demo data                 |
| schema gen  | `pnpm db:generate` after editing `src/db/schema/`     |

## Layout (`src/`)

| dir          | what                                                 |
|--------------|------------------------------------------------------|
| `app.ts`     | Fastify app build (plugins → modules → error handler)|
| `server.ts`  | entrypoint                                           |
| `modules/`   | feature plugins: `routes.ts` · `service.ts` · `repository.ts` (split into `repository/*.repo.ts` when large) |
| `adapters/`  | external IO behind interfaces: `llm/` `github/` `git/` `astgrep/` `tokenizer/` `secrets/` `codeindex/` `depgraph/` `embedder/` `auth/` `mocks.ts` |
| `platform/`  | DI container, config (`loadConfig`), error handling  |
| `db/`        | Drizzle schema (`schema/`), `migrate.ts`, `seed.ts`  |
| `prompts/`   | agent prompt templates                               |
| `vendor/shared/` | **source of truth** for Zod contracts (`@devdigest/shared`) |

## Non-default conventions

- Filenames are `kebab-case`. Feature code lives under `modules/<feature>/`.
- Routes are schema-first via `fastify-type-provider-zod` — Zod schemas in
  `vendor/shared/contracts/*.ts` drive both request validation and response
  serialization. Don't hand-roll `Schema.parse(req.body)` in handlers.
- Plugins (`helmet`, `cors`, `rate-limit`, SSE) register **before** modules so
  encapsulated module plugins inherit them.
- Modules are registered statically in `src/modules/index.ts` (one import + one
  `app.register` per module).
- Tests split by filename: `*.it.test.ts` → DB-backed (testcontainers Postgres);
  everything else → hermetic.
- Secrets are NOT in `AppConfig`. They flow through `SecretsProvider`
  (`~/.devdigest/secrets.json`, mode `0600`) with `process.env` fallback.
  `GITHUB_TOKEN` is canonical; `GITHUB_PAT` accepted as a fallback.

## Gotchas

- **Migrations are not applied on boot.** First boot needs `pnpm db:migrate`.
- **`*.it.test.ts` requires Docker** (testcontainers). They self-skip without it.
- **Rate limiting is global** at 120/min, disabled under `NODE_ENV=test`. Some
  routes have tighter per-route caps; SSE and `/health*` are exempt.
- **`REPO_INTEL_ENABLED` defaults to true** but degrades silently to diff-only
  when the repo isn't indexed yet. Per-agent `repo_intel` toggle gates enrichment.
- **Grounding gate is mandatory.** Findings without a citation that maps to a
  diff line are dropped before persist; score is recomputed from survivors.

## Do-not-touch

- `src/db/migrations/**` — generated by Drizzle Kit. Edit `src/db/schema/`, then
  `pnpm db:generate`. The pgvector extension is enabled by migration `0000`.
- `src/vendor/shared/` — this is the **source of truth**. The vendored copy in
  `client/src/vendor/shared/` is read-only and propagated from here.
